# 적극적 공격, 소극적 공격
적극적 공격(Active Attack)과 소극적 공격(Passive Attack)은 네트워크 보안에서 사용되는 언어입니다.
## 적극적 공격
적극적 공격은 악의적인 공격자가 네트워크나 시스템의 보안을 침해하는 공격을 말합니다. 예를 들면, 데이터의 변조, 중간자 공격, Dos/DDos 공격 등이 있습니다. 공격자는 목적 시스템에 대한 해킹, 서비스 거부 등의 공격을 수행하며, 시스템의 정상적인 동작을 방해합니다.
## 소극적 공격
소극적 공격은 공격자가 네트워크나 시스템에 대한 공격을 직접적으로 수행하지 않고, 정보를 수집하거나 감청하는 공격을 말합니다. 예를 들면, 패킷 스니핑, 포트 스캐닝, 트래픽 분석 등이 있습니다. 공격자는 이러한 방법을 통해 네트워크나 시스템에서 전송되는 데이터의 내용이나 목적지 등의 정보를 수집합니다.
# 정보보안의 목표 6가지
정보보안의 목표 6가지는 다음과 같다.

1. 기밀성(Confidentiality): 인가된 사람들만이 접근한 상태를 유지하는 것을 의미합니다. 기밀성은 암호화와 엑세스 제어 등의 기술적 제어와 교육 등의 비기술적 제어를 통해 유지됩니다.

2. 무결성 (Integrity): 정보가 무결하고 변조되지 않은 상태를 유지하는 것을 의미합니다. 정보의 무결성은 해시 함수, 전자 서명, 버전 관리 등의 기술적 제어와 절차적 제어 등을 통해 보장됩니다.

3. 가용성 (Availability): 필요한 사용자가 정보나 시스템에 접근 가능한 상태를 유지하는 것을 의미합니다. 가용성은 시스템의 이중화, 부하 분산, 백업 등의 기술적 제어와 재해 복구 계획, 용량 관리 등의 비기술적 제어를 통해 보장됩니다.

4. 부인방지성(Non-repudiation): 발신자나 수신자가 송수신한 정보를 부인할 수 없도록 하는 것을 의미합니다. Non-repudiation은 전자 서명, 디지털 인증서, 로그 등의 기술적 제어와 법적 계약, 인수증명서 등의 비기술적 제어를 통해 보장됩니다.

5. 인증(Authentication): 접근하려는 사용자가 자신이 주장하는 사용자임을 증명하는 것을 의미합니다. 인증은 비밀번호, 생체 인식, 인증서 등의 기술적 제어와 절차적 제어 등을 통해 보장됩니다.

6. 책임 추적성(Accountability): 조직에서 정보보안 정책을 만들고 시행하는 데 있어서 누가 책임을 질 것인지를 정확히 하는 것입니다. 즉, 조직 내에서 보안에 관한 책임과 권한을 분명하게 정하는 것입니다. 
# 바이러스의 특징
바이러스는 다음과 같은 특징을 가지고 있습니다.

1. 복제: 바이러스는 자신을 복제하여 여러 컴퓨터나 파일에 감염시키는 능력이 있습니다.

2. 침투: 바이러스는 컴퓨터 시스템에 침투하여 실행됩니다. 이를 위해 보안 취약점을 이용하거나, 사용자의 무차별적인 다운로드나 이메일 첨부 파일 등을 이용합니다.

3. 파괴: 일부 바이러스는 감염된 시스템에서 데이터나 시스템 자체를 파괴하거나 변조하는 악의적인 목적을 가지고 있습니다.

4. 숨기기: 일부 바이러스는 자신을 감추거나, 탐지를 회피하기 위해 시스템에 대한 정보를 수집합니다.

5. 전파: 일부 바이러스는 인터넷이나 네트워크를 통해 전염될 수 있습니다.

6. 확산: 바이러스는 보통 대규모로 확산될 수 있습니다. 이는 하나의 컴퓨터에서 다른 컴퓨터나 네트워크로 전파될 수 있기 때문입니다.

7. 무진단: 일부 바이러스는 감염된 시스템에서 별다른 증상을 보이지 않을 수 있습니다. 이는 사용자가 감염에 대한 경고를 받지 못하게 하거나, 감염이 발견되었을 때 제거를 어렵게 만드는 것을 의미합니다.
# 악성코드
악성코드란, 다른 프로그램을 감염시켜 사용자나 시스템에 피해를 입히는 악성 소프트웨어를 의미합니다. 주로 개인정보 유출, 시스템 마비, 금전적 피해 등을 초래하는 악성코드는 매우 다양하게 존재하며, 대표적으로는 바이러스, 웜, 트로이목마, 랜섬웨어 등이 있습니다.

* 바이러스: 자기 복제 기능을 가지고, 감염된 파일이나 시스템을 복제하여 전파됩니다.
* 웜: 컴퓨터 네트워크 상에서 자기 스스로 복사하여 전파되는 악성코드입니다.
* 트로이목마: 유용한 기능을 가장하여 사용자의 컴퓨터에 침투하여 정보를 탈취하거나 시스템을 마비시키는 악성코드입니다.
* 랜섬웨어: 파일을 암호화하거나 시스템을 제어하여 금전적 이득을 추구하는 악성코드입니다.

이 외에도, 키로거, 스파이웨어, 어드웨어, 봇넷 등 다양한 종류의 악성코드가 존재합니다. 따라서, 안티바이러스 소프트웨어를 비롯한 보안 솔루션을 통해 이러한 악성코드로부터 시스템과 데이터를 보호해야 합니다.
# 대칭키 암호/ 비대칭키 암호
## 대칭키 암호
대칭키 암호는 암호화와 복호화에 같은 비밀키를 사용하는 방식입니다. 이 방식은 암호화와 복호화 속도가 빠르지만, 비밀키를 안전하게 보관해야하고 키를 교환하는 과정에서 중간자 공격에 취약할 수 있습니다.
## 비대칭키 암호
반면에 비대칭키 암호는 암호화와 복호화에 각각 다른 키를 사용하는 방식입니다. 이 방식은 대칭키 암호보다 더욱 안전하게 메시지를 전송 할 수 있으며, 중간자 공격에 대한 취약성이 적습니다. 그러나 대칭키 암호보다 속도가 느리고, 키의 길이가 더 길어서 처리 속도가 느리다는 단점이 있습니다.

예를 들어, HTTPS는 대표적인 비대칭키 암호화 방식인 SSL/TLS 프로토콜을 사용하여 웹사이트에서 전송되는 데이터를 암호화합니다. 반면에 대부분의 파일 암호화 및 복호화 작업은 대칭키 알고리즘을 사용합니다.
# 접근 제어(Access Control)
접근 제어란, 시스템에 접근할 때, 접근 가능한 사용자나 프로세스를 제한하는 보안 기술을 의미합니다. 일반적으로, 접근 제어는 3가지 방식으로 구현됩니다.

1. 주체 기반 접근 제어 (Subject-Based Access Control, SBAC)
* 주체 기반 접근 제어는, 보호 대상에 접근할 수 있는 주체(Subject)를 미리 지정하고 이들에 대한 접근 권한을 부여하는 방식입니다. 이 방식입니다. 이 방식은 역할 기반 접근 제어와 함께 가장 많이 사용되는 방식 중 하나입니다.
2. 객체 기반 접근 제어 (Object-Based Access Control, OBAC)
* 객체 기반 접근 제어는, 보호 대상(object)에 대해 접근 권한을 미리 정의하고, 이를 요청하는 주체에 부여하는 방식입니다. 예를 들어, 파일에 대한 접근 권한이 있는 사용자는 파일에 대해 접근할 수 있지만, 파일에 대한 접근 권한이 없는 사용자는 접근할 수 없습니다.
3. 속성 기반 접근 제어(Attribute-Based Access Control, ABAC)
* 속성 기반 접근 제어는, 주체와 객체의 속성 정보를 기반으로 접근 권한을 부여하는 방식입니다. 예를 들어, 파일에 대한 접근 권한을 부여 할 때, 파일의 크기, 유형, 위치 등의 속성 정보를 고려하여 접근 권한을 결정할 수 있습니다. 이 방식은 주로 클라우드 컴퓨팅과 같은 대규모 시스템에서 사용됩니다.
# 벨 라파듈라 모델(Bell-LaPadula model)과 비바 모델(Biba model)
벨 라파듈라 모델과 비바 모델은 정보 보안의 기본 개념을 기반으로 한 접근 제어 모델입니다.
## 벨 라파듈라 모델
벨 라파듈라 모델은 비밀성 기반의 모델로, 고군분투한 사용자가 시스템 보안 규칙을 우회하는 것을 방지하기 위해 설계되었습니다. 이 모델은 사용자들을 보안 수준에 따라 분류하고, 하나의 수준에서 상위 수준의 정보에 대한 접근을 제한하는 방식으로 작동합니다. 이 모델은 기밀성을 유지하기 위한 중요한 방법론입니다.
## 비바 모델
비바 모델은 무결성 기반의 모델로, 데이터를 수정하거나 손상되는 것을 방지하기 위해 설계되었습니다. 이 모델은 사용자나 프로세스가 시스템 내에서 데이터를 변경하는 것을 방지하기 위해 작동합니다. 비밀성 기반의 접근 제어와는 달리, 비바 모델은 데이터의 무결성을 보호하고자 합니다.

이 두 가지 모델은 각각 다른 목적을 가지고 있지만, 기반에 있는 개념은 비슷합니다. 두 모델 모두 정책과 규칙을 수립하여 시스템 내에서 보안을 유지하고자 합니다.
# 해시 함수
해시 함수란 임의의 길이의 데이터를 고정된 길이의 데이터로 매핑하는 함수를 의미합니다. 이때 매핑 전의 데이터를 원본, 매핑 후에 데이터를 해시 값이라고 합니다.

해시 함수의 주요 특징은 다음과 같습니다.

1. 일방향성 (One-Way function): 해시 함수는 원본 데이터를 해시 값으로 변환하는 것은 가능하지만, 해시 값을 원본 데이터로 역추적 하는 것은 매우 어렵습니다. 즉, 해시 함수는 일방향성을 가지고 있습니다.

2. 고유성 (Uniqueness): 서로 다른 두 개의 원본 데이터가 같은 해시 값을 갖는 일, 즉 해시 충돌이 발생하는 경우를 최소화해야 합니다. 즉, 해시 함수는 고유성을 가지고 있어야 합니다.

3. 고정된 출력값 (Fixed output size): 해시 함수는 입력 데이터의 길이와 상관없이 항상 일정한 크기의 출력 값을 반환합니다. 예를 들어, SHA-256 해시 함수는 항상 256 비트의 출력 값을 반환합니다.

4. 무결성 검증 (Integrity check): 해시 함수를 이용하여 원본 데이터의 무결성을 검증할 수 있습니다. 즉, 어떤 데이터가 해시 값을 가지고 있다면, 해당 데이터가 송신자가 원래 의도한 데이터인지 여부를 확인할 수 있습니다.

5. 블록체인에서의 활용: 해시 함수는 블록체인에서 새로운 블록을 생성할 때 이전 블록의 해시 값을 포함하여 새로운 블록의 해시 값을 계산하고, 이를 통해 블록체인의 무결성을 유지할 수 있습니다.
# Brute force, Dictionary Attack
Brute force 공격과 Dictionary Attack은 인증 시스템을 공격하여 비밀번호를 알아내는 방법입니다.
## Brute force
Brute force 공격은 모든 가능한 조합의 비밀번호를 시도하는 방식으로, 모든 문자, 숫자, 기호 조합에 대해 시도합니다. 이 방식은 비밀번호의 길이가 짧을 경우에는 비교적 쉽게 비밀번호를 알아낼 수 있지만, 비밀번호가 길 경우에는 시간이 오래 걸리기 때문에 효과적이지 않을 수 있습니다.
## Dictionary Attack
Dictionary Attack은 미리 준비된 사전(dictionary) 파일을 사용하여 비밀번호를 시도하는 방식으로, 사용자가 자주 사용하는 단어, 이름, 생일 등을 포함한 단어들을 대상으로 시도합니다. 이 방식은 Brute force 공격보다 빠르게 비밀번호를 알아낼 수 있는 장점이 있습니다.
