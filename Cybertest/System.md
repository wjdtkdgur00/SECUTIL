# 커널(kernel)
**커널** 은 운영 체제(OS)의 핵심 부분으로, 하드웨어와 소프트웨어 간의 인터페이스 역할을 합니다. 운영 체제는 커널 위에 구축되어 있으며, 커널은 하드웨어를 관리하고 프로세스와 시스템 자원(메모리, 파일 시스템, 네트워크 등)에 대한 접근을 제공합니다.

커널은 보통 다음과 같은 기능을 수행합니다.
```
1. 하드웨어 추상화: 커널은 다양한 하드웨어를 추상화하여 하드웨어와 소프트웨어 간의 일관성 있는 인터페이스를 제공합니다. 이를 통해 응용 프로그램은 특정 하드웨어에 대한 지식 없이도 하드웨어를 사용할 수 있습니다.

2. 자원 관리: 커널은 프로세스가 사용하는 시스템 자원(메모리, 파일 시스템, 네트워크 등)을 관리하고, 이들 자원에 대한 접근 권한을 설정합니다.

3. 프로세스 관리: 커널은 프로세스를 생성, 삭제, 스케줄링하고, 프로세스 간 통신(IPC)을 제공합니다.

4. 보안: 커널은 시스템 자원에 대한 접근 권한을 설정하여, 보안을 유지합니다.
```
커널은 운영 체제의 핵심이므로, 운영 체제의 안정성과 성능에 직접적인 영항을 미칩니다. 따라서 운영 체제의 개발자들은 커널의 안정성과 효율성을 높이기 위해 지속적으로 최적화를 수행합니다.
# 리눅스 기본 명령어
리눅스는 다양한 명령어를 제공하여 사용자가 터미널을 통해 시스템을 조작 할 수 있습니다. 이번에는 리눅스의 기본적인 명령어에 대해서 알아보겠습니다.

1. ls : 디렉토리 내의 파일 목록을 보여줍니다.
```bash
ls
```
2. cd: 디렉토리를 변경합니다.
```bash
cd [디렉토리_경로]
```
3. pwd: 현재 작업 중인 디렉토리 경로를 출력합니다.
```bash
pwd
```
4. mkdir: 디렉토리를 생성합니다.
```arduino
mkdir [디렉토리_이름]
```
5. rmdir: 디렉토리를 삭제합니다.
```
rmdir [디렉토리_이름]
```
6. touch: 파일을 생성합니다,
```bash
touch [파일_이름]
```
7. rm: 파일을 삭제합니다.
```bash
rm [파일_이름]
```
8. cp: 파일을 복사합니다.
```css
cp [원본_파일_경로] [복사_파일_경로]
```
9. mv: 파일을 이동하거나 이름을 변경합니다.
```css
mv [원본_파일_경로] [복사_파일_경로]
```
10. cat: 파일 내용을 출력합니다.
```bash
cat [파일_이름]
```
11. echo: 문자열을 출력하거나 파일에 문자열을 씁니다.
```css
echo [문자열]
echo [문자열] > [파일_이름]
```
12. ps: 현재 실행 중인 프로세스 목록을 보여줍니다.
```
ps
```
13. kill: 프로세스를 강제로 종료합니다.
```bash
kill [프로세스_ID]
```
14. top: 시스템 상태를 모니터링합니다.
```css
top
```
15. ifconfig: 네트워크 인터페이스 정보를 출력합니다.
```
ifconfig
```
16. ping: 네트워크 연결 상태를 확인합니다.
```css
ping [호스트이름또는IP주소]
```

이 외에도 리눅스에는 수많은 명령어들이 존재합니다. 이러한 명령어들은 터미널에서 직접 입력하여 사용할 수 있습니다.
# 파일 시스템(File System)
**파일 시스템**은 **컴퓨터에서 파일과 디렉토리(폴더)를 저장하고 관리하는 체계**입니다. 파일 시스템은 하드 디스크, SSD, USB 드라이브 등의 저장장치를 관리하며 파일과 디렉토리를 구성하고 접근 권한, 속성, 크기 등의 정보를 유지합니다.

파일 시스템은 운영체제에서 제공하며, 대표적으로 Windows 운영체제에서는 NTFS, FAT, exFAT 등의 파일 시스템을 지원하고, macOS는 HFS+, APFS등 의 파일 시스템을 지원하며, 리눅스는 ext4, XFS, Btrfs 등의 파일 시스템을 지원합니다.

파일 시스템을 파일을 저장할 때, 디스크 상의 빈 영역을 찾아 파일의 내용을 쓰며, 파일의 크기와 내용이 변경될 때마다 파일 시스템은 파일의 위치와 크기를 업데이트합니다. 또한 파일 시스템은 파일과 디렉토리에 대한 메타 데이터 (metadata) 정보를 유지합니다. 메타 데이터는 파일의 이름, 크기, 소유자, 권한, 생성일, 수정일 등의 정보를 포함합니다.

이러한 파일 시스템의 기능은 파일의 안정성, 보안성, 효율성 등을 보장하는데 중요한 역할을 합니다. 파일 시스템은 운영체제와 밀접한 관련이 있으며, 운영 체제의 파일 시스템 지원 여부와 파일 시스템의 성능 및 기능은 컴퓨터 사용에 영향을 미칩니다.

# 아이노드 (inode)
아이노드는 Unix 계열 운영체제에서 파일 시스템 내의 파일이나 디렉토리를 식별하는 고유하는 고유한 식별자입니다. 아이노드는 각 파일이나 디렉토리의 메타 데이터 (metadata) 정보를 저장하며, 파일 이름, 소유자, 권한, 시간 정보 등을 포함합니다.

각 파일이나 디렉토리는 아이노드 번호와 연결되어 있으며, 파일 시스템 내의 모든 파일은 각각 고유한 아이노드 번호를 가지고 있습니다. 이를 통해 파일 시스템은 각 파일을 구별하고, 파일의 메타 데이터를 관리할 수 있습니다.

아이노드는 파일 시스템의 성능과 안정성에 큰 영향을 미치며, 아이노드의 크기는 파일 시스템의 최대 파일 크기와 파일 수를 결정합니다. 작은 아이노드 크기는 많은 수의 파일을 지원할 수 있지만, 각 파일의 메타 데이터를 저장하기에 충분한 공간이 없을 수 있습니다. 반면 큰 아이노드 크기는 메타 데이터를 충분히 저장할 수 있지만, 파일 시스템이 처리해야 할 파일 수를 제한할 수 있습니다.

리눅스에서는 ext파일 시스템에서 128바이트의 아이노드 크기를 기본으로 사용합니다. 이를 통해 대부분의 파일 시스템에서 충분한 아이노드 공간을 제공하면서도 파일 시스템의 최대 파일 수를 유지할 수 있습니다.
# 리눅스 파일권한
리눅스에서 파일 권한(permission)은 파일이나 디렉토리에 대한 접근 제어를 위해 사용됩니다. 파일 권한은 3개의 부분으로 구성되어 있으며, 각 부분은 3비트씩으로 이루어져 있습니다. 이 3개의 부분은 다음과 같습니다.
```
1. 소유자 권한: 파일이나 디렉토리를 생성한 사용자에 대한 권한입니다.

2. 그룹 권한: 파일이나 디렉토리를 생성한 사용자가 속한 그룹에 대한 권한입니다.

3. 기타사용자 권한: 소유자나 그룹에 속하지 않은 모든 사용자에 대한 권한입니다.
```
각 권한은 3비트의 이진수로 표시됩니다. 각각의 비트는 다음과 같은 의미를 가지고 있습니다.

* 4 (2의2승): 읽기 권한
* 2 (2의1승): 쓰기 권한
* 1 (2의0승): 실행 권한

파일 권한은 숫자로도 표현될 수 있습니다. 각 권한을 2의 거듭제곱 수로 변환하여 합산한 결과가 권한에 해당하는 숫자입니다. 예를 들어, 소유자가 읽기와 쓰기 권한을 가지고, 그룹과 기타 사용자는 읽기 권한만 가지고 있다면, 파일 권한은 다음과 같이 표현됩니다.

* 소유자: 6(읽기 권한 4+쓰기권한 2)
* 그룹: 4(읽기 권한만 있으므로 4)
* 기타 사용자: 4(읽기 권한만 있으므로 4)

이를 합산하여 파일 권한은 644가 됩니다. 이는 소유자는 읽기와 쓰기 권한, 그룹과 기타 사용자는 읽기 권한만 가지고 있는 것을 나타냅니다.
# 리눅스 관리자권한 명령어
리눅스에서 관리자 권한(root 권한)을 사용하면 시스템의 모든 작업으 수행할 수 있습니다. 관리자 권한으로 실행해야 하는 몇 가지 작업은 다음과 같습니다.<br>
1. 사용자 권한 
* su: 다른 사용자로 전환할 때 사용합니다.
* sudo: 일시적으로 root 권한으로 실행할 때 사용합니다.
2. 시스템관리
* systemctl: 서비스 제어를 위해 사용합니다.
* apt-get,yum등: 패키지 관리자를 사용하기 위해 사용합니다.
3. 파일 및 디렉토리 관리
* chown: 파일이나 디렉토리의 소유자를 변경할 때 사용합니다.
* chmod: 파일이나 디렉토리의 권한을 변경할 때 사용합니다.
* rm: 파일이나 디렉토리를 삭제할 때 사용합니다.
4. 시스템 정보 조회
* top: 현재 실행 중인 프로세스 정보를 조회합니다.
* ps: 프로세스 정보를 조회합니다.

위와 같은 명령어들은 일반 사용자로는 실행할 수 없고, root 권한으로 실행해야 합니다. 그러나 root 권한을 계속 사용하는 것은 보안 문제가 발생할 수 있으므로, 필요한 경우에만 일시적으로 root 권한을 얻어 작업을 수행해야 합니다. 이를 위해 sudo 명령어를 사용하는 것이 좋습니다.
# 리눅스 배포판 종류
리눅스는 여러 개의 배포판 (distribution)이 존재합니다. 각각의 배포판은 리눅스 커널을 기반으로 하며, 추가적인 소프트웨어와 설정, 사용자 인터페이스 등이 다를 수 있습니다. 주요한 리눅스 배포판에는 다음과 같은 것들이 있습니다.

1. Ubuntu: 데스크톱 및 서버 환경에서 사용하기 위한 인기 있는 배포판 중 하나입니다. 사용하기 쉽고 안정적이며, 무료로 사용할 수 있습니다.

2. Debian: Ubuntu의 기반이 되는 배포판입니다. 안정성과 보안성이 높아 서버 환경에서 널리 사용됩니다.

3. CentOS: 레드햇 엔터프라이즈 리눅스(RHEL)의 무료 클론 버전입니다. 서버 환경에서 널리 사용됩니다.

4. Fedora: 레드햇의 커뮤니티 버전으로, 새로운 기능을 빠르게 적용하고 실험하기 위한 목적으로 사용됩니다.

5. Arch Linux: 매우 경량화된 배포판으로, 최신 소프트웨어를 지원하며 매우 유연하고 가볍습니다.

6. openSUSE: 서버 및 데스크톱 환경에서 사용하기 위한 유연한 배포판입니다. 새로운 기능을 적극적으로 채택하며, KDE Plasma 데스크톱 환경을 기본으로 제공합니다.

7. Kali Linux: 해킹 및 보안 검사를 위한 특화된 배포판입니다.

이 외에도 다양한 리눅스 배포판이 존재합니다. 각각의 배포판은 서로 다른 특징을 가지고 있으므로, 사용 목적에 맞는 적절한 배포판을 선택하는 것이 중요합니다.
# 윈도우 사용자 그룹
윈도우 운영체제에서는 다음과 같은 기본적인 사용자 그룹이 존재합니다.

1. Administrators 그룹: 시스템 관리자 권한을 가진 그룹으로, 모든 작업을 수행할 수 있습니다. 로컬 컴퓨터나 도메인의 모든 컴퓨터에서 모든 리소스에 대한 접근 권한을 가지고 있습니다.

2. Users 그룹: 일반 사용자가 속한 그룹으로, 파일과 폴더에 대한 읽기와 쓰기 등의 작업을 수행할 수 있습니다.

3. Guests 그룹: 로컬 컴퓨터에 임시로 접근하는 사용자를 위한 그룹입니다. 일반적으로 Guest 계정은 비활성화되어 있습니다.

4. Power Users 그룹: 로컬 컴퓨터에서 일부 시스템 설정과 관련된 작업을 수행할 수 있는 그룹입니다. 하지만 Administrators 그룹과 같이 모든 작업을 수행할 수는 없습니다.

5. Backup Operators 그룹: 시스템 백업을 수행할 수 있는 권한을 가진 그룹입니다.

6. Remote Desktop Users 그룹: 원격 데스크톱 연결을 허용하는 사용자 그룹입니다.

이 외에도 추가적인 사용자 그룹을 만들거나, 기존 그룹의 권한을 수정하여 사용자 계정의 권한을 조정할 수 있습니다.
# 각 운영체제의 특징
다양한 운영체제가 존재하며 각각의 특징은 다음과 같습니다.

1. Windows
* 마이크로소프트사에서 개발한 운영체제로, 대중적으로 가장 많이 사용되고 있습니다.
* 그래픽 사용자 인터페이스(GUI)를 제공하며, 윈도우 응용 프로그램들을 쉽게 실행할 수 있습니다.
* Windows API를 이용하여 응용프로그램을 개발할 수 있습니다.
2. macOS
* 애플사에서 개발한 운영체제로, 맥 컴퓨터에서 사용됩니다.
* 시스템 안정성과 보안성이 높아서, 그래픽 디자인 및 개발과 같은 업무용으로 많이 사용됩니다.
* Unix 기반으로, POSIX 표준을 준수합니다.
3. Linux
* 리눅스 커널을 기반으로한 운영체제입니다. 여러개의 배포판이 존재하며, 무료로 사용할 수 있습니다.
* 유닉스와 유사한 환경을 제공하며, 다양한 용도로 사용됩니다.
* 오픈소스 소프트웨어이기 때문에, 개발자가 직접 수정하고 개발할 수 있습니다.
4. Android
* 구글에서 개발한 모바일 운영체제입니다.
* 대부분의 모바일 기기에서 사용되며, 다양한 애플리케이션을 지원합니다.
* 리눅스 기반으로, 오픈소스 소프트웨어입니다.
5. iOS
* 애플사에서 개발한 모바일 운영체제입니다.
