# IPv4 / IPv6
## IPv4
* IPv4는 인터넷의 초기 형태에서 사용되던 프로토콜입니다. IPv4 주소는 32비트로 구성되며, 점으로 구분된 4개의 숫자로 표시됩니다. 예를 들어, 192.168.0.1과 같은 형태로 표시됩니다. IPv4 주소 공간은 42억개의 주소를 가지고 있지만, 인터넷 사용량의 급격한 증가로 인해 IPv4 주소 공간이 고갈되고 있습니다.
## IPv6
* IPv6는 IPv4 주소 고갈 문제를 해결하기 위해 개발된 프로토콜입니다. IPv6 주소는 128비트로 구성되며, 16진수 8개의 그룹으로 표시됩니다. 예를 들어, 2001:0db8:85a3:0000:0000:8a2e:0370:7334와 같은 형태로 표시됩니다. IPv6 주소 공간은 약 340,282,366,920,938,463,463,374,607,431,768,211,456개의 (많다)주소를 가지고 있어 IPv4 주소 고갈 문제를 완전히 해결할 수 있습니다.

* IPv4와 IPv6는 주소 체계가 다르기 때문에 서로 호환되지 않습니다. 따라서, IPv6로 전환하기 위해서는 인프라와 네트워크 장비 등을 업그레이드 해야 합니다. 그러나, IPv6는 더 큰 주소 공간을 가지고 있으며, QoS(품질보증)와 보안 등 의 기능을 지원하기 때문에 앞으로 인터넷에서 널리 사용될 것으로 예상됩니다.
# TCP Segement 
* TCP (Transmission Control Protocol)는 인터넷에서 가장 널리 사용되는 프로토콜 중 하나로, 신뢰성 있는 데이터 전송을 보장합니다. TCP는 데이터를 세그먼트 (segement) 단위로 나누어 전송합니다.

* TCP 세그먼트는 헤더와 페이로드로 구성됩니다. 헤더에는 세그먼트를 보내는 컴퓨터와 받는 컴퓨터의 IP 주소와 포트 번호, 전송 제어 정보 등의 정보가 포함되어 있습니다. 페이로드는 전송할 데이터입니다.

* TCP 세그먼트는 MTU(Maximum Transmission Unit)보다 작아야 하며, 일반적으로 1500 바이트 이하로 설정됩니다. 이는 네트워크에서 전송하는 데이터의 크기가 일정 이상을 넘어가면 전송 속도가 느려지기 때문입니다. 따라서, 큰 데이터를 전송할 경우에는 여러 개의 TCP 세그먼트로 나누어서 전송합니다.

* TCP는 전송 중 손실된 세그먼트를 감지하고 재전송을 요청하는 등의 기능을 제공하여 데이터 전송의 신뢰성을 보장합니다. 이를 위해 각 세그먼트에는 일련 번호(sequence number)와 응답 번호(acknowledgment number)가 포함되어 있습니다. 수신자는 세그먼트를 받은 후 확인 응답 세그먼트(ACK)를 보내 송신자에게 해당 세그먼트가 제대로 전송되었음을 알리고, 손실된 세그먼트가 있다면 재전송을 요청합니다.

* 따라서, TCP는 안정적이고 신뢰성 있는 데이터 전송을 위해 세그먼트 단위로 데이터를 분할하고 전송합니다. 이르 통해 데이터 전송 중 손실이나 오류가 발생 했을 경우에도 신뢰성 있는 데이터 전송을 보장할 수 있습니다.
# 네트워크 장비
네트워크 장비는 컴퓨터 네트워크에서 사용되는 여러 종류의 하드웨어 장치로, 네트워크를 구성하고 통신을 관리하기 위해 사용됩니다. 주요한 네트워크 장비는 다음과 같습니다.

1. 허브 (Hub)
* 허브는 여러 대의 컴퓨터나 네트워크 장비를 연결하여 하나의 네트워크로 구성하는 장치입니다. 허브는 수신한 데이터를 모든 포트로 전달하여 데이터 전송의 부하를 일으킬 수 있으므로, 대부분의 경우에는 대역폭이 좁아 성능이 떨어집니다.
2. 스위치 (Switch)
* 스위치는 허브와 같이 여러 대의 컴퓨터나 네트워크 장비를 연결하여 하나의 네트워크로 구성하는 장치입니다. 하지만, 스위치는 포트간의 데이터 전송을 각각 독립적으로 처리하므로 데이터 전송의 부하가 덜 발생하여 대역 폭이 넓고 성능이 좋습니다.
3. 라우터 (Router)
* 라우터는 서로 다른 네트워크 간의 데이터 전송을 관리하는 장치입니다. 라우터는 IP 주소를 기반으로 경로를 결정하여 데이터를 전송하며, 여러 개의 네트워크를 연결할 수 있습니다.
4. 게이트웨이 (Gateway)
* 게이트웨이는 두 개 이상의 서로 다른 네트워크를 연결하는 장치입니다. 인터넷과 같은 대규모 네트워크에서는 게이트웨이가 인터넷과 사설 네트워크를 연결하고, 데이터 패킷을 다른 네트워크를 전달합니다.
5. 모뎀 (Modem)
* 모뎀은 디지털 신호를 아날로그 신호로 변환하여 전화 회선을 통해 데이터를 전송하는 장치입니다. 모뎀은 이더넷 케이블이나 광케이블과 같은 디지털 통신망에서 사용되는 장비와 연결하여 두 네트워크 간의 데이터 전송이 가능하도록 합니다.
6. 방화벽 (Firewall)
* 방화벽은 네트워크를 보호하기 위한 장치로, 네트워크의 보안 정책을 설정하고, 외부로부터의 침입을 차단하여 네트워크의 안정성을 유지합니다.

이외에도 다양한 네트워크 장비가 존재합니다.
# 무선 네트워크
* 무선 네트워크는 유선이나 케이블이 없이 무선 신호를 이용하여 컴퓨너나 다른 디바이스들을 연결하는 컴퓨터 네트워크 입니다. 일반적으로 무선 네트워크는 WI-FI라고도 불리며, 가정이나 공공 장소, 기업 등 다양한 장소에서 사용됩니다.

* 무선 네트워크는 무선 엑세스 포인트(AP)를 중심으로 동작합니다. 무선 엑세스 포인트는 무선 신호를 생성하고, 무선 클라이언트(무선 디바이스)와 통신을 하여 무선 네트워크를 구성합니다. 무선 클라이언트는 노트북, 스마트폰, 태블릿, 게임 콘솔 등 다양한 디바이스가 될 수 있습니다.

* 무선 네트워크는 주파수 대역에 따라 2.4GHz와 5GHz로 나뉘며, 각각의 대역은 장단점이 존재합니다. 2.4GHz 대역은 장거리 전송이 가능하며 벽이나 장애물에 대한 감쇠가 적지만, 대역폭이 좁고 많은 무선 디바이스가 함께 사용될 때 충돌이 발생할 수 있습니다. 반면에 5GHz 대역은 대역폭이 넓고 빠른 속도로 데이터 전송이 가능 하지만, 전파의 직진성이 강하여 벽이나 장애물에 대한 감쇠가 크고, 전송 거리가 짧습니다.
# SSH (Secure Shell)
* SSH는 네트워크 상에서 안전하게 원격지 컴퓨터에 로그인하거나 다른 네트워크 서비스를 이용할 수 있게 해주는 프로토콜입니다. SSH는 암호화 기술을 이용하여 데이터를 안전하게 전송하며, 원격지 컴퓨터에 로그인할 때 ID와 패스워드를 안전하게 전송하도록 보장합니다.

* SSH는 기본적으로 텍스트 기반의 명령어 인터페이스를 제공하며, 로그인한 후에는 쉘을 이용하여 원격지 컴퓨터를 제어할 수 있습니다. SSH는 이외에도 파일 전송, 포트 포워딩, TCP와 X11 포워딩 등 다양한 기능을 제공합니다. 또한, SSH 클라이언트와 SSH 서버가 서로 다른 운영체제에서도 작동할 수 있기 때문에, 크로스 플랫폼 환경에서도 이용이 가능합니다.
* SSH는 TCP 포트 22를 사용합니다. 보안이 강화된 SSH는 보안성이 강화된 Telnet과 같은 프로토콜이라고 할 수 있으며, 원격지 컴퓨터에 안전하게 접속하고 관리하는 데에 매우 유용합니다.
# ICMP(Internet Control Message Protocol)
* ICMP는 인터넷 프로토콜(IP)에서 사용되는 제어 메시지 프로토콜로, 네트워크 통신에서 오류 및 제어 메시지를 전달하는 데 사용됩니다. ICMP는 IP의 일부분으로, IP 데이터그램의 내부에 담겨 전송하며, 네트워크 장비나 호스트 간의 통신 문제 해결을 위해 사용됩니다.
* ICMP는 다양한 유형의 메시지를 지원합니다. 가장 많이 사용되는 유형은 Echo Request와 Echo Replay 메시지입니다. Echo Request는 일반적으로 "Ping"이라고 알려져 있으며, 호스트 간의 연결 상태를 테스트하는데 사용됩니다. 호스트에서 Echo Request 메시지를 보내면, 대상 호스트는 Echo Replay 메시지로 응답합니다. 이외에도, ICMP 메시지는 시간초과, 경로 변경, 라우터 공지, 네트워크 문제 등 다양한 문제를 식별하고 해결하는 데 사용됩니다.
* ICMP 메시지는 보통 네트워크 상태를 모니터링하고 문제를 해결하는데 사용됩니다. 예를 들어, 네트워크 장애가 발생했을 때 ICMP 메시지를 사용하여 장애의 원인을 파악하고, 문제를 해결할 수 있습니다. ICMP 메시지는 또한 네트워크 보안에서도 사용됩니다. ICMP Flood 공격은 대량의 ICMP 패킷을 목표 호스트에 보내 네트워크 서비스를 마비시키는 공격 방법 중 하나입니다.
# Dos, DDos
Dos(Denial of Service)와 DDos(Distributed Denial of Service)는 네트워크 보안에서 자주 언급되는 공격 유형입니다.
## Dos
* Dos 공격은 하나의 공격자가 한 대의 컴퓨터를 사용하여 대상 서버나 네트워크에 대량의 트래픽을 보내고, 서버가 이를 처리하지 못하도록 하는 공격입니다. 이러한 공격으로 서비스 거부(Denial of Service)가 발생하며, 대상 서버나 네트워크에서는 다른 유저들도 서비스 거부 상태가 되어 일시적인 서비스 마비 현상이 발생할 수 있습니다.
## DDos
* DDos 공격은 Dos 공격과 유사하지만, 여러 대의 컴퓨터나 장치를 사용하여 대상 서버나 네트워크에 대량의 트래픽을 보냅니다. 이러한 공격에서는 다수의 컴퓨터나 장치를 이용하기 때문에 Dos 공격보다 대량의 트래픽이 발생하며, 대상 서버나 네트워크에서 이를 처리하지 못하게 됩니다. DDos 공격은 분산된 방식으로 이루어지기 때문에, 공격 대상을 특정하는 것이 어렵고, 대처도 어려울 수 있습니다.
# Spoofing
* Spoofing은 컴퓨터 네트워크에서 보안 공격의 한 종류로, IP주소, MAC주소 등을 조작하여 다른 컴퓨터나 장치에서 보내는 것처럼 위장하여 데이터를 보내는 것입니다.

* 예를 들어, IP Spoofing은 공격자가 자신의 IP 주소 대신 다른 컴퓨터의 IP 주소를 사용하여 데이터를 보내는 것을 말합니다. 이러한 Spoofing 공격으로 인해 공격 대상 컴퓨터나 네트워크에서는 위조된 IP 주소에서 데이터가 전송된 것으로 인식하게 되어, 보안상의 문제가 발생할 수 있습니다.

* MAC Spoofing은 네트워크에서 통신할 때 사용되는 MAC 주소를 위조하여, 다른 컴퓨터나 장치의 MAC 주소를 사용하여 데이터를 보내는 것입니다. 이를 이용하여, 공격자는 다른 사용자의 컴퓨터로부터 정보를 도용하거나, 네트워크의 보안을 우회하여 침입하는 등의 악의적인 행위를 할 수 있습니다.
* Spoofing 공격을 막기 위해서는, 인증 절차, 암호화, 방화벽 등의 보안 장치를 이용하여 네트워크 보안을 강화하고, IP 주소나 MAC 주소를 확인하는 등의 보안 검사를 수행해야 합니다. 또한, 보안 취약점을 발견하면 이를 즉시 수정하여 보안성을 강화해야 합니다.
# Firewall(방화벽)
* Firewall은 컴퓨터와 컴퓨터 사이, 또는 컴퓨터와 인터넷 사이에서 데이터 통신을 제어하는 보안 장치입니다. 일반적으로 Firewall은 네트워크 상에서 허용되지 않은 접근을 차단하고, 허용된 접근만을 통과시킴으로써, 컴퓨터 및 네트워크의 보안을 강화합니다.

* Firewall은 주로 패킷 필터링, NAT(Network Address Translation), 프록시 서버 등의 방식으로 동작합니다. 패킷 필터링은 데이터 패킷의 출발지, 목적지, 포트 등의 정보를 검사하여, 설정된 규칙에 따라 허용 또는 차단하는 방식입니다. NAT는 내부망에서 사용하는 사설 IP 주소를 외부망에서 사용하는 공인 IP주소로 변환하여, 외부로부터 내부망을 보호하는 방식입니다. 프록시 서버는 사용자의 요청을 대신하여 외부 서버와 통신하여, 사용자의 실제 IP 주소를 감추는 등의 보안 기능을 제공합니다.
